# FrameDrop 设计文档 — 视频抽帧定格动画工具

## 概述

FrameDrop 是一个 Python 桌面应用，将视频抽帧转换为定格动画效果。用户导入视频后，通过调整抽帧间隔和目标帧率，实时预览定格动画效果，并导出为 MP4 或 GIF。

## 技术栈

- **GUI**: PySide6
- **帧提取**: OpenCV (opencv-python)
- **导出**: FFmpeg (系统安装，subprocess 调用)
- **Python**: 3.9+

## 架构

### 项目结构

```
FrameDrop/
├── main.py              # 入口
├── ui/
│   ├── main_window.py   # 主窗口
│   └── preview_widget.py # 视频预览组件
├── core/
│   ├── extractor.py     # 帧提取 (OpenCV)
│   └── exporter.py      # 视频导出 (FFmpeg)
└── requirements.txt
```

### 数据流

1. **导入** — 用户选择视频文件 → OpenCV 读取元信息（时长、原始帧率、总帧数）
2. **抽帧** — 根据用户设置的帧率，OpenCV 提取对应帧 → 保存到临时目录
3. **预览** — 按顺序播放缓存的帧图片，模拟定格动画效果
4. **导出** — FFmpeg 将帧序列合成 MP4 或 GIF

## UI 设计

### 布局

- **左侧 (70%)**: 视频预览区域
  - 未导入时显示"拖拽或点击导入视频"
  - 导入后自动循环播放定格动画预览
- **右侧 (30%)**: 控制面板
  - 选择视频按钮 + 当前文件名
  - 抽帧间隔滑块 (每 N 帧取 1 帧，范围 1-30，默认 2)
  - 目标帧率滑块 (FPS，范围 1-30，默认 8)
  - 导出格式选择 (MP4 / GIF)
  - 导出按钮
  - 进度条
- **底部状态栏**: 当前状态 | 原始视频信息 | 抽帧后帧数

### 交互逻辑

| 操作 | 行为 |
|------|------|
| 点击/拖拽视频 | 导入视频，读取元信息，自动按默认参数抽帧 |
| 调整抽帧间隔 | 重新抽帧并刷新预览 |
| 调整目标帧率 | 改变预览播放速度 |
| 导出 | 后台执行，显示进度条，完成后提示保存位置 |

## 核心处理逻辑

### 帧提取

输入视频通过 OpenCV 的 `VideoCapture` 逐帧读取，根据抽帧间隔 N 跳帧，将选中的帧保存为 PNG 到临时目录 `temp/framedrop/{timestamp}/`。

预览使用 QTimer 按目标帧率（如 8 FPS = 125ms/帧）逐帧加载图片到 QLabel，循环播放。

### 导出

| 格式 | FFmpeg 命令 |
|------|-------------|
| MP4 | `ffmpeg -r {fps} -i frame_%04d.png -c:v libx264 -pix_fmt yuv420p output.mp4` |
| GIF | `ffmpeg -r {fps} -i frame_%04d.png -vf "fps={fps},split[s0][s1];[s0]palettegen[p];[s1][p]paletteuse" output.gif` |

### 临时文件管理

- 启动时创建 `temp/framedrop/{timestamp}/`
- 导出完成后保留缓存（方便调参重新预览）
- 退出程序时自动清理

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 不支持的视频格式 | 导入时检测，弹窗提示支持格式（MP4/AVI/MOV/MKV） |
| 视频文件过大 | 抽帧在 QThread 中执行，不卡死 UI |
| FFmpeg 未安装 | 启动时检测，提示安装 |
| 导出中途取消 | 取消按钮停止 FFmpeg 进程，清理临时文件 |
| 重复调参 | 复用已缓存帧，只重新抽取变化部分 |
| 程序异常退出 | 下次启动清理残留 temp 目录 |
